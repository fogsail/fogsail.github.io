C. Directional Increase
题目大意，一开始给你一个长度为 nnn 的数组，最开始的元素都只有 000，指针指在第 111 个位置
两种操作任意次
第一种，如果指针不在 nnn，可以将当前指针指向的元素 +1+1+1，然后指针移动到下一个位置
第二种，如果指针不在 111，可以将当前指针指向的元素 −1-1−1，然后指针移动到前一个位置
给你一个数组 aaa，判断 aaa 能不能由任意次两种操作其中一种，变换而成
注意变换完成之后，指针必须重新指向 111 这个位置

算法分析
从后往前贪心，将 aaa 数组的后缀 000 去掉，得到 bbb 数组，令此时 bbb 数组长度为 nnn

很关键的是，变换完之后指针必须重新指向 111 这个位置，由此出发，我们考虑从 bbb 数组能不能逆变换成 <0,0⋯ ,0>\left<0,0\cdots, 0 \right>⟨0,0⋯,0⟩
突破口不难想到，必须有 bn<0b_n < 0bn​<0，因为指针要回退到 111 这个位置，所以 bnb_nbn​ 只能往前走若干次
bnb_nbn​ 初始时候是 000，构造 bnb_nbn​ 之后指针回退到位置 111，所以 bnb_nbn​ 只会不断减小

下面考虑构造，(bn−1−∣bn∣,0)→(bn−1,0)→(bn−1,−∣bn∣)(b_{n-1}-|b_n|, 0) \to (b_{n-1}, 0) \to (b_{n-1}, -|b_n|)(bn−1​−∣bn​∣,0)→(bn−1​,0)→(bn−1​,−∣bn​∣)
上面的过程是指针从 n−1n-1n−1 处向右移动 111 个单位，移动 ∣bn∣|b_n|∣bn​∣ 次，这样指针移动完在 nnn 位置
然后再从 nnn 位置向左移动，回退 ∣bn∣|b_n|∣bn​∣ 次，指针又回到 n−1n-1n−1，恰好是一次合法的构造

也就是说，从后往前贪心，如果遇到 bi+1<0b_{i+1} < 0bi+1​<0，逆变换 bi=bi−∣bi+1∣b_{i} = b_{i} - |b_{i+1}|bi​=bi​−∣bi+1​∣，使得 bi+1b_{i+1}bi+1​ 这一位为 000
并且 bib_ibi​ 形成新的后缀，并且 bi<0b_i < 0bi​<0 才能继续做下去

注意到 bi=0b_i = 0bi​=0 只有当 i=1i = 1i=1 时候成立才可以继续做，因为你指针必须回退到 111
最后当指针回退到 111 的时候判一下 b[1]b[1]b[1] 是否为 000 即可

I like Query Problem
题目大意，给你长度为 nnn 的序列和 qqq 个询问
(1,l,r,x)(1, l, r, x)(1,l,r,x) 表示将 [l,r][l, r][l,r] 的数都改成 ⌊aix⌋\displaystyle \left\lfloor \frac{a_i}{x} \right\rfloor⌊xai​​⌋
(2,l,r,y)(2, l, r, y)(2,l,r,y) 将 [l,r][l, r][l,r] 的数都改成 yyy
(3,l,r)(3, l, r)(3,l,r) 求 [l,r][l, r][l,r] 的和

算法分析
首先对于区间赋值，[l,r]←v[l, r] \leftarrow v[l,r]←v 很简单，就是标准的线段树操作

难点是在于 ⌊aid⌋\displaystyle\left\lfloor \frac{a_i}{d} \right\rfloor⌊dai​​⌋

对于这种修改，最坏情况可能有 222 种

  • [l,k],[k+1,r][l, k], [k+1, r][l,k],[k+1,r] 两个区间的值可能不同
  • [l,k][l, k][l,k] 可能执行了 222 次，⌊aid2⌋\displaystyle \left\lfloor \frac{a_i}{d^2} \right\rfloor⌊d2ai​​⌋
    [k+1,r][k+1, r][k+1,r] 可能执行了 333 次，⌊aid3⌋\displaystyle \left\lfloor \frac{a_i}{d^3} \right\rfloor⌊d3ai​​⌋

不论哪种情况，对第一种操作而言，都可能存在 [l,r][l, r][l,r] 区间值不统一的情况
对于这种情况，可以用 segment tree beats 来实现，具体来说

  • 对于 [l,r][l, r][l,r] 执行 ai←ai/xa_i \leftarrow a_i / xai​←ai​/x 的操作，如果 [l,r][l, r][l,r] 中的所有元素都相同，不妨设都是 vvv
    此时相当于区间赋值操作，直接给区间打赋值标记 v/xv / xv/x
  • 否则的话，暴力递归所有子节点，直到找到满足所有数都相同的区间，打标记 v/xv / xv/x
    然后回溯的时候，更新路径上的值
  • 至于一个区间是否所有数都相同？可以维护一个最大值 mxmxmx，一个最小值 mnmnmn，当 mx=mnmx = mnmx=mn 时候区间所有数都相同
    另外还需要维护区间的 sum,szsum, szsum,sz

segment tree beats 的时间复杂度平均是 O((n+q)log⁡n)O((n+q) \log n)O((n+q)logn)

不相邻的组合数个数
从 nnn 个球中选出 rrr 个不相邻的球的方案

  • n<2r−1n < 2r -1n<2r−1 不存在这样的方案，答案是 000
  • n⩾2r−1n \geqslant 2r-1n⩾2r−1 时候，取法是 (n−r+1r)\displaystyle \binom{n-r+1}{r}(rn−r+1​)
    这怎么来理解呢？
    首先有 rrr 个球，中间有 r−1r-1r−1 个空格，这 r−1r-1r−1 个数只能看成是空格而不是球
    于是我们取的球只能从 n−r+1n-r+1n−r+1 个里面取，任意取 rrr 个，然后将之前的空格插入到 r−1r-1r−1 个空位中
    (n−r+1r)\displaystyle \binom{n-r+1}{r}(rn−r+1​) 种方案数

那么 nnn 个球围成一圈呢？

  • 注意到一个球周围至少要有 222 个空格，所以 n<2rn < 2rn<2r 时无解
  • n⩾2rn \geqslant 2rn⩾2r，对球进行编号，对于所有可能的组合，都只有 222 种情况
    包含 111 号球和不包含 111 号球
    如果包含 111 号球，那么 (n−1,1,2)(n-1, 1, 2)(n−1,1,2) 这 333 个位置不能选，所以只能从 n−3n-3n−3 个球中选
    从 n−3n-3n−3 个球中选出 r−1r-1r−1 个不相邻的球的方案数是 ((n−3)−(r−1)+1r−1)=(n−r−1r−1)\displaystyle \binom{(n-3)-(r-1) + 1}{r-1} = \binom{n-r-1}{r-1}(r−1(n−3)−(r−1)+1​)=(r−1n−r−1​)
    如果不包含 111 号球，可以从 n−1n-1n−1 个数中任选 rrr 个不相邻的球，方案数是 ((n−1)−r+1r)=(n−rr)\displaystyle \binom{(n-1)-r+1}{r} = \binom{n-r}{r}(r(n−1)−r+1​)=(rn−r​)
